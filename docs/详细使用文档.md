# Dia-Agent 详细使用文档

本文档面向项目使用者与开发者，覆盖从环境准备、图谱与 RAG 构建、服务启动、评测验证到常见故障排查的完整流程。

## 1. 项目定位与核心能力

Dia-Agent 是一个 **Guardrail-First（红线优先）** 的糖尿病专病智能体。其核心目标是：

- 在输出诊疗建议前，先基于图谱执行硬约束检索。
- 将结构化红线（禁忌、阈值、剂量调整）与指南检索结果共同输入推理。
- 通过审计节点确保输出不违反禁忌。

系统主流程：

`Perception -> Guardrail -> Reasoner -> Auditor`

## 4. 配置项详解

通过环境变量 `DIA_AGENT_*` 配置系统。

### 4.1 Neo4j（结构化约束）

```bash
export DIA_AGENT_NEO4J_URI="bolt://localhost:7687"
export DIA_AGENT_NEO4J_USER="neo4j"
export DIA_AGENT_NEO4J_PASSWORD="your-password"
export DIA_AGENT_NEO4J_DATABASE="neo4j"
```

### 4.2 推理 LLM（OpenAI 兼容协议）

```bash
export DIA_AGENT_LLM_BASE_URL="https://your-llm-endpoint/v1"
export DIA_AGENT_LLM_API_KEY="your-api-key"
export DIA_AGENT_LLM_MODEL="Qwen/Qwen2.5-7B-Instruct"
export DIA_AGENT_LLM_WIRE_API="chat_completions"
```

### 4.3 视觉模型（可选，多模态感知）

```bash
export DIA_AGENT_VISION_BASE_URL="https://your-vision-endpoint/v1"
export DIA_AGENT_VISION_API_KEY="your-vision-api-key"
export DIA_AGENT_VISION_MODEL="Qwen/Qwen2-VL-7B-Instruct"
export DIA_AGENT_VISION_WIRE_API="chat_completions"
```

### 4.4 回退策略

- 未配置 Neo4j 密码：自动回退 `dataset/graph.json`。
- 未配置 LLM：Reasoner 使用模板策略输出。
- 未配置视觉模型：Perception 对图片输入退化为文本规则解析。

## 5. 构建知识图谱（Neo4j）

### 5.1 常规导入

```bash
python scripts/import_graph_to_neo4j.py \
  --input dataset/graph.json \
  --uri bolt://localhost:7687 \
  --user neo4j \
  --password "your-password" \
  --database neo4j
```

### 5.2 清库后重建（危险操作）

```bash
python -c 'from neo4j import GraphDatabase; d=GraphDatabase.driver("bolt://localhost:7687",auth=("neo4j","your-password")); s=d.session(database="neo4j"); s.run("MATCH (n) DETACH DELETE n").consume(); s.close(); d.close()'

python scripts/import_graph_to_neo4j.py \
  --input dataset/graph.json \
  --uri bolt://localhost:7687 \
  --user neo4j \
  --password "your-password" \
  --database neo4j \
  --skip-constraints
```

说明：当库内已有同字段索引时，可能出现约束冲突，此时可使用 `--skip-constraints`。

### 5.3 导入后核验

```bash
python -c 'from neo4j import GraphDatabase; d=GraphDatabase.driver("bolt://localhost:7687",auth=("neo4j","your-password")); s=d.session(database="neo4j"); print("Node:", list(s.run("MATCH (n) UNWIND labels(n) AS l RETURN l, count(*) c ORDER BY l"))); print("Rel:", list(s.run("MATCH ()-[r]->() RETURN type(r) t, count(*) c ORDER BY t"))); s.close(); d.close()'
```

## 6. 构建 RAG 向量索引

### 6.1 使用默认两份指南 PDF

```bash
python scripts/build_rag_index.py --reset
```

### 6.2 自定义 PDF/集合名/持久化目录

```bash
python scripts/build_rag_index.py \
  --pdf dataset/中国1型糖尿病诊治指南（2021版）.pdf \
  --pdf dataset/dc26sint.pdf \
  --persist-dir data/chroma \
  --collection dia_guidelines \
  --embedding-model moka-ai/m3e-base \
  --reset
```

## 7. 启动与使用

### 7.1 CLI 问诊（结构化输入）

```bash
python scripts/run_consultation.py \
  --raw-input '{"indicators":{"eGFR":25,"HbA1c":9.3},"diseases":["糖尿病酮症酸中毒"],"current_drugs":["盐酸二甲双胍片 (普通片/缓释片)"]}' \
  --rag-query "1型糖尿病 肾功能不全 用药"
```

### 7.2 CLI 问诊（文本 + 图片）

```bash
python scripts/run_consultation.py \
  --raw-input "患者eGFR:25，HbA1c:9.3，病史糖尿病酮症酸中毒" \
  --image data/uploads/lab_001.png \
  --rag-query "1型糖尿病 肾功能不全 用药"
```

### 7.3 启动 API

```bash
python scripts/run_api.py
```

默认地址：`http://127.0.0.1:8000`

健康检查：

```bash
curl http://127.0.0.1:8000/health
```

问诊请求（结构化）：

```bash
curl -X POST "http://127.0.0.1:8000/v1/consult" \
  -H "Content-Type: application/json" \
  -d '{
    "raw_input": {
      "indicators": {"eGFR": 25, "HbA1c": 9.3},
      "diseases": ["糖尿病酮症酸中毒"],
      "current_drugs": ["盐酸二甲双胍片 (普通片/缓释片)"]
    },
    "rag_query": "1型糖尿病 肾功能不全 用药",
    "history_text": ""
  }'
```

问诊请求（多模态路径输入）：

```bash

  -H "Content-Type: application/json" \
  -d '{

      "image_paths": ["data/uploads/lab_001.png"],
```

```bash
streamlit run dia_agent/ui/streamlit_app.py
```

### 8.1 50 个陷阱案例评测（推荐）

## 10. 返回结果结构说明

`/v1/consult` 返回核心字段：

- `patient_state`：标准化后的患者状态
- `guardrail_report`：禁忌命中与剂量调整
- `reasoner_result`：最终建议、推荐药物、引用片段
- `audit_result`：是否通过审计、违规项
- `trace`：工作流执行轨迹

## 11. 常见问题（Troubleshooting）

### 11.1 Neo4j 认证失败

症状：`Neo.ClientError.Security.Unauthorized`

处理：

- 核对 `DIA_AGENT_NEO4J_USER` / `DIA_AGENT_NEO4J_PASSWORD`
- 确认 Neo4j 服务已启动且端口正确

### 11.2 约束冲突（IndexAlreadyExists）

症状：创建约束时报已有索引

处理：

- 使用 `--skip-constraints` 导入
- 或先清理旧索引后再导入

### 11.3 RAG 构建慢或首次加载耗时长

说明：首次加载 embedding 模型会下载和初始化，属于正常现象。

### 11.4 使用代理时报 `socksio` 缺失

```bash
pip install socksio
```
